\section{Архитектура программного комплекса}\label{sec:software/c4}
В данной главе описывается программная реализация численных методов, представленных в~главе~\ref{ch:algorithm}, для задачи линейного раскроя в целлюлозно-бумажном производстве.
Последовательно рассматриваются архитектура программного комплекса, соответствие классов реализации сущностям математической модели (глава~\ref{ch:model}), форматы данных, пользовательский интерфейс, тестирование и вычислительные эксперименты.
Программный комплекс зарегистрирован в Реестре программ для ЭВМ (приложение~\ref{app:certificates}).

Архитектура программного комплекса описывается на трёх уровнях в нотации C4: контекст, контейнеры и компоненты.
Реализация выполнена на языке Python.

\section{Программная реализация алгоритмов}\label{sec:software/algorithms}

Численные методы из главы~\ref{ch:algorithm} реализованы в виде набора функций и классов с едиными контрактами.
Центральным объектом решения является класс \texttt{GlobalPlan}, инкапсулирующий план производства~$\overline{\overline{X}}$, план нарезки~$\overline{\overline{Y}}$ и расписание~$\overline{\overline{Z}}$.
Метод \texttt{scores()} возвращает тройку критериев $(\omega(G),\,\varphi(G),\,\tau(\overline{\overline{Y}}))$, а перегруженные операторы сравнения реализуют лексикографический порядок.
Метод \texttt{is\_valid} проверяет допустимость плана: ограничения по ширине и ножам, выполнение заказов, соблюдение дедлайнов, ограничения складов.
Единый контракт валидности используется как в генетическом поиске, так и в полном переборе, что исключает дублирование логики.

Генетический поиск (раздел~\ref{sec:alg/genetic_search}) реализован в модуле \texttt{algo/genetic\_search.py}.
Начальная популяция формируется генератором \texttt{random\_global\_plan} (раздел~\ref{sec:alg/random_plan}), глобальная мутация~--- функцией \texttt{global\_mutate}, парный отбор~--- функцией \texttt{crossingover}.
После мутации автоматически пересобираются планы производства и расписание для сохранения согласованности компонент глобального плана.

Полный перебор (раздел~\ref{sec:alg/full_enum}) реализован в модуле \texttt{algo/full\_enumeration.py} и используется как эталон для задач малой размерности.

Формирование планов производства (раздел~\ref{sec:alg/production_plans}) выполняется модулем \texttt{algo/pmm\_plans\_builder.py}, построение расписания (раздел~\ref{sec:alg/schedule})~--- модулем \texttt{algo/event\_queue\_builder.py}.
Расписание пересчитывается как при первичной генерации, так и после мутаций.

Переключение между методами осуществляется через аргумент командной строки или элемент GUI; размер популяции для генетического поиска задаётся параметром \texttt{initial\_population\_size} или берётся из \texttt{config.py}.

\section{Форматы ввода и вывода}\label{sec:software/io}

Программный комплекс поддерживает два формата входных данных.
CSV-формат используется для минимальных экспериментов; каждая строка содержит тип параметра, идентификатор и значение (пример приведён на рис.~\ref{fig:input_csv} приложения~\ref{app:progrma_gui}).

Excel-формат применяется для практической работы и содержит листы:
\begin{itemize}
    \item \textit{Slitting Machines} --- параметры ПРС/БРС;
    \item \textit{Paper Making Machines} --- параметры БДМ;
    \item \textit{Orders} --- перечень заказов;
    \item \textit{Conditions} --- кромка, время старта, марки;
    \item \textit{Storages} --- склады и ограничения.
\end{itemize}

Выходной отчёт формируется в формате CSV или Excel и включает распределение заказов по оборудованию, раскройные комбинации, потери ширины, число перестановок ножей и агрегированные показатели (пример на рис.~\ref{fig:output_xlsx} приложения~\ref{app:progrma_gui}).
При наличии дедлайнов добавляется лист расписания.

\section{Интерфейс пользователя}\label{sec:software/ui}

Программа предоставляет два варианта взаимодействия: командную строку (CLI) и графический интерфейс (GUI).

CLI (\texttt{main.py}) принимает аргументы через \texttt{argparse}:
\begin{itemize}
  \item \texttt{input\_file} --- путь к входному файлу (Excel или CSV);
  \item \texttt{output\_file} --- путь к выходному файлу;
  \item \texttt{--method} --- метод решения: \texttt{genetic\_search} (по умолчанию) или \texttt{full\_enumeration};
  \item \texttt{--input\_format}, \texttt{--output\_format} --- форматы файлов;
  \item \texttt{--initial\_population\_size} --- размер начальной популяции.
\end{itemize}
Пример запуска:
\begin{verbatim}
python main.py data/input.xlsx results/output.xlsx \
  --method genetic_search --initial_population_size 100
\end{verbatim}

GUI реализован на Tkinter (\texttt{gui.py}) и содержит поля выбора входного и выходного файлов, выбор метода решения, поле размера популяции и кнопку запуска.
Расчёт выполняется в отдельном потоке для сохранения отзывчивости интерфейса.
Примеры экранных форм приведены в приложении~\ref{app:progrma_gui} (рис.~\ref{fig:main_gui},~\ref{fig:main_in_progress}).

Оба варианта используют общий код решения: одни и те же функции для чтения данных, запуска алгоритмов и сохранения результатов.

\section{Используемые библиотеки}\label{sec:software/libs}

Реализация опирается на стандартную библиотеку Python и пакеты:

\section{Тестирование}\label{sec:software/tests}

Тестирование программного комплекса включает три уровня:
\begin{enumerate}
    \item модульные тесты --- корректность отдельных классов и функций (проверка раскроев, планов, ограничений);
    \item интеграционные тесты --- полный цикл «ввод $\rightarrow$ расчёт $\rightarrow$ вывод» для различных сценариев (без дедлайнов, с дедлайнами, с БРС, с разными марками и плотностями);
    \item экспериментальная валидация --- сравнение результатов генетического поиска с оптимальными решениями полного перебора на задачах малой размерности.
\end{enumerate}

Полная спецификация тестов приведена в приложении~\ref{app:testing-specification}.

Таким образом, эксперименты подтверждают два практических вывода: полный перебор целесообразен как эталон для задач малой размерности, а генетический поиск обеспечивает лучшее масштабирование и позволяет получать решения эталонного качества на более сложных экземплярах за приемлемое время.



\section{Выводы по главе}\label{sec:software/conclusion}

В данной главе описана программная реализация численных методов решения задачи линейного раскроя для целлюлозно-бумажного производства.

Основные результаты главы:
\begin{enumerate}
    \item Разработана модульная архитектура программного комплекса, включающая алгоритмический модуль, модельный слой и модуль ввода/вывода.
    Сопоставление сущностей математической модели и классов реализации обеспечивает прозрачность перехода от формальной постановки к программному коду.
    \item Реализованы два режима оптимизации: полный перебор (эталон для задач малой размерности) и генетический поиск (для задач средней и большой размерности).
    Единый контракт валидности и сравнения планов исключает дублирование логики между режимами.
    \item Программный комплекс поддерживает форматы CSV и Excel для ввода и вывода, а также предоставляет командный (CLI) и графический (GUI) интерфейсы.
    \item Тестирование включает модульные, интеграционные тесты и экспериментальную валидацию.
    Спецификация тестов приведена в приложении~\ref{app:testing-specification}.
    \item На примере задачи малой размерности продемонстрирована корректность работы комплекса и исследовано влияние размера популяции на качество решений.
\end{enumerate}

Программный комплекс зарегистрирован в Реестре программ для ЭВМ (приложение~\ref{app:certificates}) и может использоваться как основа для внедрения методов оптимального планирования на предприятиях целлюлозно-бумажной промышленности.
