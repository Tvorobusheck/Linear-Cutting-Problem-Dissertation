\chapter{Численные методы}\label{ch:algorithm}

В данной главе описаны численные методы решения задачи, сформулированной в главе~\ref{sec:model/statement}.
Сначала приводится метод полного перебора, используемый как эталон для задач малой размерности.
Затем описаны вспомогательные алгоритмы формирования компонент глобального плана~$G$: плана нарезки~$\overline{\overline{Y}}$, плана производства~$\overline{\overline{X}}$ и расписания~$\overline{\overline{Z}}$.
На основе этих алгоритмов строится процедура генерации случайного допустимого плана, составляющая начальную популяцию генетического поиска.
Завершает главу описание самого генетического алгоритма с~операторами мутации и~отбора, а~также анализ вычислительной сложности всех методов.

Обозначения соответствуют модели главы~\ref{sec:model/statement}: $O=\{o_1,\ldots,o_Q\}$~--- множество заказов, $M=\{m_1,\ldots,m_I\}$~--- БДМ, $C=\{c_1,\ldots,c_J\}$~--- ПРС/БРС, $T$~--- склады, $R=\sum_{i=1}^{Q}q_i$~--- суммарное число рулонов.

\section{Метод полного перебора}\label{sec:alg/full_enum}

Для целей сравнения и проверки результатов реализован метод полного перебора.
Несмотря на~экспоненциальную сложность, наличие точного метода является необходимым элементом методологии вычислительных экспериментов: он позволяет установить оптимальные значения целевой функции на~задачах малой размерности и~тем самым количественно оценить качество приближённых решений генетического алгоритма \cite{delorme2016bin}.
Кроме того, полный перебор служит средством верификации: совпадение результатов точного и~приближённого методов на~тестовых экземплярах подтверждает корректность реализации.
В контексте комбинаторной оптимизации использование полного перебора как эталона для малых экземпляров соответствует общим подходам к оценке качества эвристик \cite{papadimitriou1998_combinatorial_opt, scheithauer2017_cutpack_optimization}.

Его идея состоит в~рекурсивном построении всех возможных распределений заказов по~ПРС и~выборе наилучшего плана по~целевой функции~$\mu(G)$.

В~отличие от~единой рекурсии, алгоритм разделён на две фазы, обусловленные наличием «малых» заказов (рулоны, нарезаемые на~БРС).
На~первой фазе распределяются малые заказы $O^{\mathrm{small}}=\{o\in O:\,g_o=1\}$ по~станкам с~признаком ${g^c}_j=1$.
После их распределения собираются слитые мастер-рулоны~$\mathcal{S}$, порождённые малыми раскроями.
На~второй фазе распределяются обычные заказы $O^{\mathrm{norm}}=O\setminus O^{\mathrm{small}}$ и~слитые мастер-рулоны~$\mathcal{S}$ по~станкам с~${g^c}_j=0$.

На~каждом шаге рекурсии выбирается не~полностью распределённый заказ, и~для~каждого совместимого станка выполняется попытка добавить рулон процедурой $\mathrm{AddPlanRoll}$ (см.\ раздел~\ref{sec:alg/cutting_plans}).
Если добавление успешно, рекурсия продолжается; из~всех допустимых завершений выбирается лучшее по~$\mu(G)$.
По завершении обеих фаз вызываются алгоритмы формирования плана производства~$\overline{\overline{X}}$ и~расписания~$\overline{\overline{Z}}$ (разделы~\ref{sec:alg/production_plans},~\ref{sec:alg/schedule}), после чего проверяется допустимость итогового плана.

Пошаговое описание алгоритма полного перебора:
\begin{enumerate}
  \item Инициализировать пустой глобальный план $G_0$ с~$J$ пустыми планами нарезки.
  \item Если среди заказов есть малые ($O^{\mathrm{small}}\neq\varnothing$), перейти к~фазе~1; иначе~--- к~фазе~2 с~$\mathcal{S}=\varnothing$.
  \item \textit{Фаза~1 (малые заказы).} Для каждого невыполненного малого заказа $o\in O^{\mathrm{small}}$ и~каждого станка $c_j$ с~${g^c}_j=1$: создать копию плана~$G'$, попытаться добавить рулон вызовом $\mathrm{AddPlanRoll}(j,o)$.
  При~успехе рекурсивно продолжить фазу~1 с~$G'$.
  Из~всех допустимых завершений сохранить лучшее по~$\mu(G)$.
  \item После распределения всех малых заказов собрать слитые мастер-рулоны~$\mathcal{S}$ из~сформированных раскроев БРС.
  \item \textit{Фаза~2 (обычные заказы и~$\mathcal{S}$).} Аналогично фазе~1, но~для~обычных заказов $O^{\mathrm{norm}}$ и~слитых мастер-рулонов~$\mathcal{S}$ на~станках с~${g^c}_j=0$.
  \item Когда все заказы распределены и~$\mathcal{S}=\varnothing$, вызвать $\mathrm{Finalize}(G)$: сгенерировать план производства~$\overline{\overline{X}}$, расписание~$\overline{\overline{Z}}$ и~проверить допустимость плана.
  \item Вернуть лучший допустимый план по~$\mu(G)$ среди всех перебранных ветвей.
\end{enumerate}

Процедура $\mathrm{Finalize}(G)$ генерирует план производства~$\overline{\overline{X}}$ и расписание~$\overline{\overline{Z}}$, после чего проверяет допустимость полного плана; если план недопустим, возвращается пустой результат.

Сложность метода определяется числом ветвлений.
Обозначим:
\[
R_s=\sum_{o\in O^{\mathrm{small}}}q_o,\quad R_n=\sum_{o\in O^{\mathrm{norm}}}q_o,\quad J_s=|\{j:g^c_j=1\}|,\quad J_n=|\{j:g^c_j=0\}|.
\]
Первая фаза порождает до~$J_s^{R_s}$ ветвей, вторая~--- до~$J_n^{R_n+|\mathcal{S}|}$.
Общая оценка:
\[
  T_{\mathrm{full}} = O\!\bigl(J_s^{R_s}\cdot J_n^{R_n+|\mathcal{S}|}\bigr) \le O(J^R),
\]
где $J=\max(J_s,J_n)$, $R$~--- суммарное число рулонов.
Каждое ветвление включает глубокое копирование плана~$O(R)$ и~вызов $\mathrm{AddPlanRoll}$~$O(n_{\max}^c)$, что даёт полиномиальный множитель.
Метод пригоден только для задач малой размерности и~используется как базовый эталон \cite{vanderbeck2000integer,delorme2016bin}.

\section{Формирование плана нарезки $\overline{\overline{Y}}$}\label{sec:alg/cutting_plans}

Центральной подзадачей при построении глобального плана является распределение рулонов по~станкам и~формирование раскроев.
Для каждого ПРС $c_j$ план нарезки $\overline{Y}_j$ представляет собой последовательность раскроев $Y_{j,1},Y_{j,2},\ldots$, каждый из~которых содержит набор форматов $y_{j,k,\ell}$ с~шириной ${r^Y}_{j,k,\ell}$ и~список мастер-рулонов.

В~классических постановках задачи раскроя \cite{gilmore1961linear,gilmore1963linear} схемы раскроя формируются заранее, а~оптимальные объёмы их использования определяются методами линейного программирования.
Однако в~рассматриваемой модели каждый раскрой порождает мастер-рулоны, привязанные к~конкретному ПРС (с~учётом типа станка, марки, плотности и~числа слоёв), что делает множество допустимых схем зависимым от~порядка назначения заказов на~станки.
Поэтому вместо полного перечисления схем используется инкрементальная стратегия: рулоны добавляются по~одному процедурой $\mathrm{AddPlanRoll}$, которая автоматически формирует раскрои и~мастер-рулоны с~учётом ограничений по~ширине~${w^c}_j$, числу ножей~${n^c}_j$ и~кромке~$V$.
Такой подход, относящийся к~семейству конструктивных жадных эвристик \cite{cerqueira2021,haessler1991, yagiura2017_cutpack_combinatorial}, обеспечивает линейную сложность по~числу рулонов и~гарантирует выполнение ограничений модели на~каждом шаге.

Алгоритм $\mathrm{AddPlanRoll}$ добавляет один рулон в~план нарезки $j$-го станка.
Сначала предпринимается попытка вставить рулон в~последний раскрой $Y_{j,|\overline{Y}_j|}$; если это невозможно (нарушается ограничение по~ширине или числу ножей), создаётся новый раскрой.
При добавлении первого формата в~раскрой формируются мастер-рулоны: для малого заказа~--- слитые мастер-рулоны типа $\mathrm{SlittedMasterRoll}$ шириной ${w^c}_j$, для обычного~--- стандартные мастер-рулоны.
Число мастер-рулонов равно числу слоёв~$l_o$ заказа.

Пошаговое описание алгоритма $\mathrm{AddPlanRoll}(j, o)$:
\begin{enumerate}
  \item Если план нарезки станка~$c_j$ содержит хотя~бы один раскрой ($|\overline{Y}_j|>0$), попытаться добавить рулон~$o$ в~последний раскрой $Y_{j,|\overline{Y}_j|}$ вызовом $\mathrm{AddLayoutRoll}$.
  \item Если добавление в~последний раскрой успешно, вернуть $\mathrm{True}$.
  \item Иначе создать новый пустой раскрой $L_{\mathrm{new}}$ и~добавить его в~конец $\overline{Y}_j$.
  \item Попытаться добавить рулон~$o$ в~$L_{\mathrm{new}}$ вызовом $\mathrm{AddLayoutRoll}$.
  \item Если успешно, вернуть $\mathrm{True}$; иначе удалить $L_{\mathrm{new}}$ из~$\overline{Y}_j$ и~вернуть $\mathrm{False}$.
\end{enumerate}

\end{center}

Пошаговое описание алгоритма $\mathrm{AddLayoutRoll}(L, o, c_j)$:
\begin{enumerate}
  \item Если раскрой $L$ пуст ($|L.\mathrm{rolls}|=0$):
  \begin{enumerate}
    \item Если ширина рулона $w_o > {w^c}_j$, вернуть $\mathrm{False}$ (рулон шире станка).
    \item Если заказ малый ($g_o=1$), создать $l_o$ слитых мастер-рулонов шириной~${w^c}_j$.
    \item Иначе создать $l_o$ стандартных мастер-рулонов шириной~${w^c}_j$.
  \end{enumerate}
  \item Добавить рулон~$o$ в~список форматов раскроя $L.\mathrm{rolls}$.
  \item Проверить ограничения: $V + \sum_k {r^Y}_{j,k} \le {w^c}_j$ и~$|L.\mathrm{rolls}| \le {n^c}_j$.
  \item Если оба ограничения выполнены, вернуть $\mathrm{True}$.
  \item Иначе удалить рулон~$o$ из~$L.\mathrm{rolls}$ и~вернуть $\mathrm{False}$.
\end{enumerate}

Сложность одного вызова $\mathrm{AddPlanRoll}$ составляет $O({n^c}_j)$, где ${n^c}_j$~--- максимальное число ножей (определяет верхнюю границу числа форматов в~раскрое).
Последовательное добавление всех $R=\sum q_i$ рулонов требует $O(R\cdot n^c_{\max})$ операций.

\textbf{Пример.}
Пусть ПРС $c_1$ имеет ${w^c}_1=60$, ${n^c}_1=3$, кромка $V=1$, а~план нарезки $\overline{Y}_1$ пока содержит один раскрой $Y_{1,1}$ с~единственным форматом $y_{1,1,1}=(40,\,4)$ (ширина~$40$, заказ~$o_4$).
Требуется добавить рулон заказа $o_3$ с~$w_3=30$, $l_3=1$, $g_3=0$.

\begin{enumerate}
\item $|\overline{Y}_1|=1>0$, поэтому вызывается $\mathrm{AddLayoutRoll}(Y_{1,1},\,o_3,\,c_1)$.
\item Раскрой $Y_{1,1}$ не~пуст ($|Y_{1,1}.\mathrm{rolls}|=1$), формат $o_3$ добавляется: $Y_{1,1}.\mathrm{rolls}=\{(40,4),\,(30,3)\}$.
\item Проверка ограничений: $V+40+30=71>60={w^c}_1$~--- ширина превышена. Формат удаляется; $\mathrm{AddLayoutRoll}$ возвращает $\mathrm{False}$.
\item Создаётся новый раскрой $Y_{1,2}$. Поскольку $Y_{1,2}$ пуст и~$w_3=30\le 60={w^c}_1$, формируется $l_3=1$ мастер-рулон шириной~$60$.
Формат $(30,3)$ добавляется: $Y_{1,2}.\mathrm{rolls}=\{(30,3)\}$.
Проверка: $V+30=31\le 60$ и~$|Y_{1,2}.\mathrm{rolls}|=1\le 3={n^c}_1$.
$\mathrm{AddPlanRoll}$ возвращает $\mathrm{True}$.
\end{enumerate}
В~итоге $\overline{Y}_1=\bigl(Y_{1,1}:\{(40,4)\};\;Y_{1,2}:\{(30,3)\}\bigr)$.

\section{Формирование плана производства $\overline{\overline{X}}$}\label{sec:alg/production_plans}

После построения планов нарезки $\overline{\overline{Y}}$ необходимо определить, какие тамбуры и~на~каких БДМ следует произвести.
В~традиционных постановках задачи раскроя \cite{haessler1991,gilmore1961linear} план производства тамбуров не~рассматривается как отдельная подзадача, поскольку предполагается, что полуфабрикат доступен в~неограниченном количестве.
В~рассматриваемой модели, напротив, тамбуры производятся на~конкретных БДМ с~ограничениями по~ширине~${w^m}_i$, и~назначение тамбуров на~машины влияет на~допустимость расписания и~загрузку складов; подобные связи между раскроем и~планированием выпуска партий характерны для интегрированных производственных постановок \cite{pochet2006_prodplan_mip, melega2018review}.
Алгоритм $\mathrm{GenerateProductionPlan}$ обеспечивает автоматическое построение плана производства, согласованного с~планом нарезки, что является необходимым связующим звеном в~интегрированной модели.

Каждый раскрой $Y_{j,k}$ на~станке $c_j$ с~${g^c}_j=0$ содержит список мастер-рулонов $Y_{j,k}.\mathrm{master\_rolls}$, каждый из~которых характеризуется шириной, маркой и~плотностью.
Для каждого такого мастер-рулона определяется множество совместимых БДМ:
\[
M_{\mathrm{comp}}(u) = \bigl\{\,m_i\in M : |{w^m}_i - w_u| < \varepsilon\,\bigr\},
\]
и~случайным образом выбирается одна из~них.
Мастер-рулон добавляется в~план производства выбранной БДМ.

\resizebox{!}{0.8\textheight}{%
    \input{Dissertation/images/generate_pm_plans.tikz}
}
\captionof{figure}{Блок-схема формирования плана производства $\overline{\overline X}$}
\label{alg:generate_pm_plans}
\end{center}

Пошаговое описание алгоритма $\mathrm{GenerateProductionPlan}$:
\begin{enumerate}
  \item Инициализировать пустые планы производства $\overline{X}_i=\varnothing$ для каждой БДМ $m_i\in M$.
  \item Для каждого станка $c_j$ с~${g^c}_j=0$ и~каждого раскроя $L\in\overline{Y}_j$:
  \begin{enumerate}
    \item Для каждого мастер-рулона $u\in L.\mathrm{master\_rolls}$:
    \begin{enumerate}
      \item Определить множество совместимых БДМ $M_{\mathrm{comp}}(u)=\{m_i\in M:|{w^m}_i-w_u|<\varepsilon\}$.
      \item Случайным образом выбрать БДМ $m_i\in M_{\mathrm{comp}}(u)$.
      \item Добавить мастер-рулон~$u$ в~план производства $\overline{X}_i$.
    \end{enumerate}
  \end{enumerate}
  \item Вернуть набор планов производства $(\overline{X}_1,\ldots,\overline{X}_I)$.
\end{enumerate}

Пусть
\[
N_Y=\sum_{j=1}^{J}|\overline{Y}_j|
\]
--- суммарное число раскроев, $L_{\max}$~--- максимальное число слоёв в~одном раскрое.
Тогда сложность алгоритма составляет
\[
O(N_Y \cdot L_{\max} \cdot I),
\]
где множитель~$I$ обусловлен поиском совместимых БДМ.

\textbf{Пример.}
Пусть $M=\{m_1,m_2\}$ с~${w^m}_1=40$, ${w^m}_2=60$, а~план нарезки содержит два станка ($c_1$, $c_2$) с~${g^c}_1={g^c}_2=0$.
Раскрой $Y_{1,1}$ на~$c_1$ (${w^c}_1=60$) имеет марку ${f^Y}_{1,1}=1$, плотность ${p^Y}_{1,1}=100$ и~$l_{1,1}=1$ слой; ему соответствует один мастер-рулон $u_1$ шириной~$60$.
Раскрой $Y_{2,1}$ на~$c_2$ (${w^c}_2=40$) имеет марку ${f^Y}_{2,1}=1$, плотность ${p^Y}_{2,1}=100$ и~$l_{2,1}=1$ слой; мастер-рулон~$u_2$ шириной~$40$.

\begin{enumerate}
\item Инициализируются пустые планы: $\overline{X}_1=\varnothing$, $\overline{X}_2=\varnothing$.
\item Для $u_1$ (ширина~$60$): $M_{\mathrm{comp}}(u_1)=\{m_2\}$, поскольку ${w^m}_2=60$. Выбирается $m_2$; $\overline{X}_2=\{X_{2,1}=(60,1,100)\}$.
\item Для $u_2$ (ширина~$40$): $M_{\mathrm{comp}}(u_2)=\{m_1\}$, поскольку ${w^m}_1=40$. Выбирается $m_1$; $\overline{X}_1=\{X_{1,1}=(40,1,100)\}$.
\end{enumerate}
Результат: $\overline{\overline{X}}=\bigl(\overline{X}_1:\{(40,1,100)\};\;\overline{X}_2:\{(60,1,100)\}\bigr)$.

\section{Формирование расписания $\overline{\overline{Z}}$}\label{sec:alg/schedule}

При наличии временных ограничений необходимо построить расписание $\overline{\overline{Z}}$, определяющее последовательность и~моменты выполнения операций производства и~нарезки.
Каждая операция $\overline{Z}=(h,\overline{h},e,d,z,\overline{T},\overline{\overline{T}},\overline{t}')$ фиксирует время начала~$h$ и~окончания~$\overline{h}$, тип~$e\in\{0,1\}$ (производство/раскрой), машину~$d$, индекс плана~$z$ и~снимки состояния складов.

Задача построения расписания для~многомашинной системы с~промежуточными складами существенно отличается от~классического планирования нарезки, где временные и~складские аспекты не~рассматриваются \cite{gilmore1961linear,haessler1991}; общие методы календарного планирования многомашинных систем изложены в \cite{pinedo2023_scheduling, brucker2007_scheduling_algorithms}.
В~интегрированных постановках \cite{keskinocak2002,nonas2000combined,westerlund1998trimloss_scheduling} расписание связывает этапы выпуска и~нарезки: операция нарезки может начаться только при~наличии подходящих тамбуров на~складе, а~производство тамбура ограничено ёмкостью склада.
Разработанный алгоритм $\mathrm{GenerateEventQueue}$ симулирует эту связь через расписание с~двумя типами складов~--- текущим~$\overline{T}$ и~запланированным~$\overline{\overline{T}}$,~--- что позволяет учитывать как реальное, так и~будущее состояние полуфабрикатов и~корректно определять доступность машин.

Алгоритм использует текущее время~$t$, счётчики выполненных операций $z_c$ и~$x_m$ для каждой машины, текущее состояние склада~$\overline{T}$ и~будущее~$\overline{\overline{T}}$, а~также очередь с~приоритетом~$\mathcal{E}$ (min-heap по времени окончания).

На~каждом шаге определяются множества доступных машин:
\begin{align*}
  C_{\mathrm{avail}}(t) &= \bigl\{c_j : z_j < |\overline{Y}_j|,\;\lambda_j \le t,\;\overline{T}\text{ содержит тамбуры для }Y_{j,z_j+1}\bigr\},\\
  M_{\mathrm{avail}}(t) &= \bigl\{m_i : x_i < |\overline{X}_i|,\;\lambda'_i \le t,\;\overline{\overline{T}}\text{ может вместить }X_{i,x_i+1}\bigr\},
\end{align*}
где $\lambda_j$, $\lambda'_i$~--- моменты окончания последних операций на~соответствующих машинах.

Если хотя~бы одна машина доступна, случайным образом выбирается тип операции и~машина; операция назначается с~текущим временем~$t$, вычисляется её длительность и~обновляется состояние складов.
Если ни~одна машина не~доступна, из~очереди $\mathcal{E}$ извлекается ближайшая завершающаяся операция, время сдвигается к~$\overline{h}$ этой операции, и~склады обновляются.

}
\captionof{figure}{Блок-схема формирования расписания}
\label{alg:generate_event_queue}
\end{center}

Пошаговое описание алгоритма $\mathrm{GenerateEventQueue}$:
\begin{enumerate}
  \item Инициализировать текущее время $t=H$, счётчики $z_j=0$, $x_i=0$, моменты окончания $\lambda_j=H$, $\lambda'_i=H$ для всех машин; склады $\overline{T}=\varnothing$, $\overline{\overline{T}}=\varnothing$; очередь $\mathcal{E}=\varnothing$; историю $\overline{\overline{Z}}=\varnothing$.
  \item Пока существуют невыполненные операции или $\mathcal{E}\neq\varnothing$:
  \begin{enumerate}
    \item Определить множества доступных ПРС $C_{\mathrm{avail}}(t)$ и~БДМ $M_{\mathrm{avail}}(t)$.
    \item Если $C_{\mathrm{avail}}\neq\varnothing$ или $M_{\mathrm{avail}}\neq\varnothing$:
    \begin{enumerate}
      \item Случайным образом выбрать тип операции (производство или нарезка) и~машину.
      \item Если выбрана нарезка на~$c_j$: удалить мастер-рулоны раскроя $Y_{j,z_j+1}$ из~$\overline{T}$ и~$\overline{\overline{T}}$, вычислить длительность~$\tau$, добавить операцию $(t,t+\tau,1,j,z_j,\overline{T},\overline{\overline{T}})$ в~$\mathcal{E}$, обновить $\lambda_j=t+\tau$, $z_j=z_j+1$.
      \item Если выбрано производство на~$m_i$: вычислить длительность~$\tau$, добавить операцию $(t,t+\tau,0,i,x_i,\overline{T},\overline{\overline{T}})$ в~$\mathcal{E}$, обновить $\lambda'_i=t+\tau$, $x_i=x_i+1$, поместить тамбур $X_{i,x_i}$ в~$\overline{\overline{T}}$.
    \end{enumerate}
    \item Иначе (ни одна машина не~доступна):
    \begin{enumerate}
      \item Извлечь из~$\mathcal{E}$ операция~$Z$ с~наименьшим временем окончания, сдвинуть $t=\overline{h}(Z)$, добавить~$Z$ в~$\overline{\overline{Z}}$.
      \item Если $e(Z)=0$ (производство): переместить тамбур из~$\overline{\overline{T}}$ в~$\overline{T}$.
      \item Если $e(Z)=1$ (нарезка): добавить произведённые рулоны в~$\overline{T}$ и~$\overline{\overline{T}}$.
    \end{enumerate}
  \end{enumerate}
  \item Вернуть историю операций $\overline{\overline{Z}}$.
\end{enumerate}

Пусть
\[
N_{\mathrm{ops}}=\sum_{i=1}^{I}|\overline{X}_i|+\sum_{j=1}^{J}|\overline{Y}_j|
\]
--- общее число операций.
Каждая итерация основного цикла выполняет: определение доступных машин~$O(I+J)$, операцию с~очередью~$O(\log N_{\mathrm{ops}})$ и~обновление складов~$O(K)$.
Итоговая сложность алгоритма:
\[
T_{\mathrm{sched}} = O\bigl(N_{\mathrm{ops}}\cdot(I+J+\log N_{\mathrm{ops}}+K)\bigr).
\]

\textbf{Пример.}
Пусть $I=2$, $J=1$, $K=2$, $H=\text{08:00}$, ${v^m}_1=\text{0:06}$, ${v^c}_1=\text{1:30}$, ${k^c}_1=\text{0:48}$.
Планы: $\overline{X}_1=\{X_{1,1}=(40,1,100)\}$, $\overline{X}_2=\{X_{2,1}=(60,1,100)\}$; $\overline{Y}_1=\{Y_{1,1}\}$ с~$l_{1,1}=1$ слоем и~$\sigma(1,1)=2$ перестановками ножей.
Склады пусты: $\overline{T}=\overline{\overline{T}}=\varnothing$.

\begin{enumerate}
\item $t=\text{08:00}$. Определяются $C_{\mathrm{avail}}=\varnothing$ (нет тамбуров на~складе), $M_{\mathrm{avail}}=\{m_1,m_2\}$ (оба плана не~пусты, склады могут вместить тамбуры).
Случайно выбрано производство на~$m_1$: $\overline{h}=\text{08:00}+\text{0:06}=\text{08:06}$; операция $(08\!:\!00,\,08\!:\!06,\,0,\,1,\,1)$ помещается в~$\mathcal{E}$; тамбур $X_{1,1}$ добавляется в~$\overline{\overline{T}}$.
\item $t=\text{08:00}$. $M_{\mathrm{avail}}=\{m_2\}$. Производство на~$m_2$: $\overline{h}=\text{08:06}$; тамбур $X_{2,1}$ добавляется в~$\overline{\overline{T}}$.
\item $t=\text{08:00}$. $C_{\mathrm{avail}}=\varnothing$, $M_{\mathrm{avail}}=\varnothing$. Извлекается операция $m_1$ с~$\overline{h}=\text{08:06}$; $t\leftarrow\text{08:06}$; тамбур $X_{1,1}$ перемещается из~$\overline{\overline{T}}$ в~$\overline{T}$.
\item $t=\text{08:06}$. Теперь $C_{\mathrm{avail}}=\{c_1\}$ (тамбур доступен). Выбрана нарезка на~$c_1$: тамбур $X_{1,1}$ удаляется из~$\overline{T}$; $\overline{h}=\text{08:06}+\text{1:30}\cdot 1+\text{0:48}\cdot 2=\text{10:12}$; операция помещается в~$\mathcal{E}$.
\end{enumerate}
Процесс продолжается до~исчерпания всех операций.

\section{Формирование случайного глобального плана}\label{sec:alg/random_plan}

Для инициализации генетического поиска требуется построить случайный допустимый план $G=(\overline{\overline{X}},\overline{\overline{Y}},\overline{\overline{Z}})$.
Качество начальной популяции существенно влияет на~сходимость генетического алгоритма \cite{goldberg1989genetic,gladkov2010, eiben2015_intro_ec}: если начальные планы разнообразны и~при~этом допустимы, эволюционный поиск быстрее обнаруживает области с~хорошими значениями целевой функции.
В~рассматриваемой задаче генерация случайного допустимого плана нетривиальна, поскольку план~$G$ состоит из~трёх взаимосвязанных компонент ($\overline{\overline{X}}$, $\overline{\overline{Y}}$, $\overline{\overline{Z}}$), каждая из~которых должна удовлетворять ограничениям модели.
Простое случайное заполнение структур данных, как правило, порождает недопустимые решения; поэтому необходим конструктивный алгоритм, выстраивающий компоненты плана в~определённом порядке и~обеспечивающий их согласованность.

Алгоритм последовательно формирует план нарезки, затем план производства и, при~необходимости, расписание.

Заказы разделяются на~малые $O^{\mathrm{small}}$ и~обычные $O^{\mathrm{norm}}$.
Внутри каждой группы заказы перемешиваются функцией $\mathrm{ShuffleByBrand}$: заказы разбиваются по~маркам, перемешиваются внутри каждой марки, затем объединяются в~порядке следования марок.
Это сохраняет технологический порядок марок, но~вносит случайность.

Каждый рулон вставляется в~план нарезки процедурой $\mathrm{AddRollRandomly}$: формируется множество совместимых станков $\{j:g^c_j=g_o\}$, порядок станков перемешивается, и~для первого станка, на~котором $\mathrm{AddPlanRoll}$ успешен, рулон фиксируется.

После распределения малых заказов собираются слитые мастер-рулоны~$\mathcal{S}$ из~раскроев БРС.
Далее обычные заказы и~$\mathcal{S}$ объединяются, перемешиваются и~распределяются аналогично.
По~завершении вызываются $\mathrm{GenerateProductionPlan}$ и, если задан временной горизонт, $\mathrm{GenerateEventQueue}$.
\begin{center}
\resizebox{!}{0.8\textheight}{%
    \input{Dissertation/images/random_global_plan.tikz}
}
\captionof{figure}{Блок-схема формирования случайного глобального плана}
\label{alg:random_global_plan}
\end{center}

Пошаговое описание алгоритма $\mathrm{RandomGlobalPlan}$:
\begin{enumerate}
  \item Инициализировать пустые планы нарезки $\overline{Y}_j=\varnothing$ для каждого станка $c_j$, $j=1,\ldots,J$.
  \item Выделить малые заказы $O^{\mathrm{small}}=\{o\in O:g_o=1\}$ и~перемешать их функцией $\mathrm{ShuffleByBrand}$.
  \item Для каждого малого заказа $o\in O^{\mathrm{small}}$ и~каждого из~$q_o$ рулонов: вызвать $\mathrm{AddRollRandomly}(\overline{\overline{Y}},o)$.
  \item Собрать слитые мастер-рулоны $\mathcal{S}$ из~раскроев станков с~${g^c}_j=1$.
  \item Сформировать множество $O^*=O^{\mathrm{norm}}\cup\mathcal{S}$ и~перемешать его функцией $\mathrm{ShuffleByBrand}$.
  \item Для каждого элемента $o\in O^*$:
  \begin{enumerate}
    \item Если $o$~--- слитый мастер-рулон: вызвать $\mathrm{AddRollRandomly}(\overline{\overline{Y}},o)$.
    \item Иначе: для каждого из~$q_o$ рулонов вызвать $\mathrm{AddRollRandomly}(\overline{\overline{Y}},o)$.
  \end{enumerate}
  \item Сформировать план производства $\overline{\overline{X}}=\mathrm{GenerateProductionPlan}(\overline{\overline{Y}},M)$.
  \item Если задан временной горизонт: сформировать расписание $\overline{\overline{Z}}=\mathrm{GenerateEventQueue}(G)$.
  \item Вернуть глобальный план $G=(\overline{\overline{X}},\overline{\overline{Y}},\overline{\overline{Z}})$.
\end{enumerate}

Сложность формирования одного случайного плана складывается из~распределения рулонов, формирования плана производства и~формирования расписания:
\[
T_{\mathrm{plan}} = O(R\cdot J\cdot n^c_{\max}) + O(N_Y\cdot L_{\max}\cdot I) + O\bigl(N_{\mathrm{ops}}\cdot(I+J+\log N_{\mathrm{ops}})\bigr).
\]
Для начальной популяции размера~$P$ суммарные затраты составляют $O(P\cdot T_{\mathrm{plan}})$.

\textbf{Пример.}
Пусть $O=\{o_1,\ldots,o_5\}$, причём $o_1$ --- малый заказ ($g_1=1$, $w_1=5$, $f_1=1$, $q_1=1$, $l_1=2$), а~$o_2,\ldots,o_5$~--- обычные ($g_i=0$).
Марки: $f_1=f_3=f_4=1$, $f_2=f_5=2$.
Станки: $c_1$ (${w^c}_1=60$, ${g^c}_1=0$), $c_2$ (${w^c}_2=40$, ${g^c}_2=0$), $c_3$ (${w^c}_3=20$, ${g^c}_3=1$).

\begin{enumerate}
\item Инициализируются пустые планы нарезки $\overline{Y}_1=\overline{Y}_2=\overline{Y}_3=\varnothing$.
\item $O^{\mathrm{small}}=\{o_1\}$. Функция $\mathrm{ShuffleByBrand}$ оставляет порядок неизменным (одна марка).
\item Для $o_1$ ($q_1=1$): $\mathrm{AddRollRandomly}$ определяет совместимые станки $\{c_3\}$ ($g^c_3=1$).
Вызов $\mathrm{AddPlanRoll}(3,o_1)$ создаёт раскрой $Y_{3,1}$ с~форматом $(5,1)$; формируются $l_1=2$ слитых мастер-рулона шириной~$20$.
\item Собираются слитые мастер-рулоны $\mathcal{S}=\{s_1,s_2\}$ (два мастер-рулона из~$Y_{3,1}$, ширина~$20$, марка~$1$, плотность~$100$).
\item $O^*=\{o_3,o_4,s_1,s_2,o_2,o_5\}$. После $\mathrm{ShuffleByBrand}$: внутри марки~$1$ порядок $\{o_3,o_4,s_1,s_2\}$ перемешивается, допустим, в~$\{o_4,s_1,o_3,s_2\}$; марка~$2$: $\{o_2,o_5\}$ допустим в~$\{o_5,o_2\}$.
\item Последовательно распределяются элементы. Например, $o_4$ ($w_4=40$, $q_4=1$) назначается на~$c_1$; слитый $s_1$ ($w=20$) назначается на~$c_1$ (добавляется в~тот~же раскрой, если позволяет ширина) или на~$c_2$, и~т.\,д.
\item Вызывается $\mathrm{GenerateProductionPlan}$ для формирования $\overline{\overline{X}}$.
\item Вызывается $\mathrm{GenerateEventQueue}$ для формирования $\overline{\overline{Z}}$.
\item Возвращается $G=(\overline{\overline{X}},\overline{\overline{Y}},\overline{\overline{Z}})$.
\end{enumerate}
Результирующий план $G$ является допустимым: все заказы распределены, ограничения по ширине и~ножам соблюдены, тамбуры назначены на~совместимые БДМ, а~расписание учитывает доступность полуфабрикатов на~складах.

\section{Операторы генетического поиска}\label{sec:alg/operators}

Генетический алгоритм использует три оператора: локальную мутацию $\mathrm{Mutate}$, глобальную мутацию $\mathrm{GlobalMutate}$ и~оператор отбора $\mathrm{Crossingover}$.

Проектирование операторов для~задач раскроя с~многокомпонентным представлением решения требует учёта специфики предметной области.
В~стандартных генетических алгоритмах \cite{goldberg1989genetic, holland1992_adaptation, goldberg1989_ga_search} мутация изменяет отдельные гены хромосомы, а~кроссинговер обменивает фрагменты между двумя хромосомами.
Однако в~рассматриваемой задаче «хромосома»~--- это глобальный план $G=(\overline{\overline{X}},\overline{\overline{Y}},\overline{\overline{Z}})$, компоненты которого жёстко связаны: изменение плана нарезки~$\overline{\overline{Y}}$ требует пересчёта плана производства~$\overline{\overline{X}}$ и~расписания~$\overline{\overline{Z}}$.
Прямой обмен фрагментами между двумя планами, как правило, нарушает эти связи и~порождает недопустимые решения \cite{kureichik1998,genetics_2}.
Поэтому операторы адаптированы к~структуре модели: мутация выполняет перестановки внутри~$\overline{\overline{Y}}$, сохраняя состав заказов, а~согласованность компонент восстанавливается пересчётом $\overline{\overline{X}}$ и~$\overline{\overline{Z}}$.

Оператор $\mathrm{Mutate}$ получает на~вход план нарезки $\overline{Y}_j$ одного станка.
Он создаёт копию плана, случайным образом перемешивает форматы $y_{j,k,\ell}$ внутри каждого раскроя $Y_{j,k}$, а~затем перемешивает порядок самих раскроев.
Перестановки не~меняют набор выполненных заказов, но~влияют на~число перестановок ножей~$\tau(\overline{\overline{Y}})$, поскольку оно зависит от~порядка форматов в~смежных раскроях.

  \node[flowproc, below=of copy] (sh1) {Для каждого раскроя\\$Y_{j,k}\in newY$\\перемешать форматы\\$Y_{j,k}.\mathrm{rolls}$};
  \node[flowproc, below=of sh1] (sh2) {Перемешать порядок\\раскроев в $newY$};
  \node[flowstop, below=of sh2] (ret) {Вернуть $newY$};
  \draw[flowline] (s) -- (copy) -- (sh1) -- (sh2) -- (ret);

Сложность $\mathrm{Mutate}$: $O(|\overline{Y}_j|+R_j)$, где $R_j$~--- суммарное число форматов в~плане станка~$j$.

Оператор $\mathrm{GlobalMutate}$ применяет $\mathrm{Mutate}$ к~каждому из~$J$ планов нарезки, после чего пересоздаёт план производства~$\overline{\overline{X}}$ и~расписание~$\overline{\overline{Z}}$ для~обеспечения согласованности компонент плана.
Новый план принимается только если $\mu(G')<\mu(G)$; в~противном случае возвращается исходный план.
Это гарантирует, что мутация не~ухудшает решение.

\begin{center}
\begin{tikzpicture}[node distance=8mm and 12mm, font=\small]
  \node[flowstart] (s) {Старт\\GlobalMutate$(G)$};
  \node[flowproc, below=of s] (copy) {Скопировать глобальный план:\\$newG\leftarrow G$};
  \node[flowproc, below=of copy] (mut) {Для каждого $j=1,\ldots,J$\\заменить $newG.Y_j$ на\\Mutate$(newG.Y_j)$};
  \node[flowproc, below=of mut] (prod) {Пересоздать план производства\\$newG.X=\mathrm{GenerateProductionPlan}(newG.Y,M)$};
  \node[flowproc, below=of prod] (sched) {Пересоздать расписание\\$newG.Z=\mathrm{GenerateEventQueue}(newG)$};
  \node[flowdec, below=of sched] (cmp) {$\mu(newG)<\mu(G)$?};
  \node[flowstop, below left=16mm and 20mm of cmp] (ret1) {Вернуть $newG$};
  \node[flowstop, below right=16mm and 20mm of cmp] (ret2) {Вернуть $G$};
  \draw[flowline] (s) -- (copy) -- (mut) -- (prod) -- (sched) -- (cmp);

\label{alg:global_mutate}

Сложность $\mathrm{GlobalMutate}$:
\[
O(R + T_X + T_Z),
\]
где
\[
T_X=O(N_Y\cdot L_{\max}\cdot I),\quad T_Z=O\bigl(N_{\mathrm{ops}}\cdot(I+J+\log N_{\mathrm{ops}})\bigr)
\]
--- стоимости формирования плана производства и~расписания соответственно.

Оператор $\mathrm{Crossingover}$ реализован как парный отбор: из~двух планов выбирается лучший по~$\mu(G)$.
Классический кроссинговер (обмен частями хромосом) не~применяется, поскольку он порождает недопустимые решения из-за~нарушения связей между компонентами $\overline{\overline{X}}$, $\overline{\overline{Y}}$ и~$\overline{\overline{Z}}$ \cite{kureichik1998,genetics_2}.

\label{alg:crossingover}
\end{center}


Сложность $\mathrm{Crossingover}$: $O(1)$ (сравнение двух векторов фиксированной размерности).

\textbf{Пример ($\mathrm{Mutate}$).}
Пусть план нарезки $\overline{Y}_1$ ПРС $c_1$ содержит три раскроя:
$Y_{1,1}:\{(40,4),\,(20,2)\}$, $Y_{1,2}:\{(50,5)\}$, $Y_{1,3}:\{(20,2),\,(20,2)\}$.
Число перестановок ножей для~исходного порядка: $\sigma(1,1)=3$, $\sigma(1,2)=3+(1-0)=4$, $\sigma(1,3)=4+(2-0)=6$; итого $\tau_1=6$.

Оператор $\mathrm{Mutate}$ перемешивает форматы внутри каждого раскроя и~порядок раскроев.
Допустим, после перестановки получен план:
$Y'_{1,1}:\{(20,2),\,(20,2)\}$, $Y'_{1,2}:\{(40,4),\,(20,2)\}$, $Y'_{1,3}:\{(50,5)\}$.
Теперь: $\sigma(1,1)=3$, $\sigma(1,2)=3+(2-1)=4$ (первый формат $(20,2)$ совпал), $\sigma(1,3)=4+(1-0)=5$; итого $\tau'_1=5<6$.
Перестановка уменьшила число переналадок на~$1$.

\textbf{Пример ($\mathrm{Crossingover}$).}
Пусть $\mu(G_1)=\langle 104,\,4,\,10\rangle$ и~$\mu(G_2)=\langle 110,\,3,\,8\rangle$.
Лексикографическое сравнение: $\omega(G_1)=104<110=\omega(G_2)$, поэтому $\mu(G_1)<\mu(G_2)$ и~оператор возвращает~$G_1$.

\section{Алгоритм генетического поиска}\label{sec:alg/genetic_search}

Задачи раскроя и~планирования производства традиционно решаются методами линейного и~целочисленного программирования \cite{haessler1991,martina2022}.
Однако при~большом числе технологических ограничений (несколько типов оборудования, переналадки, складские лимиты, многослойные заказы) размерность моделей растёт экспоненциально \cite{sgp_ccp2021_cutting_stock_modes,mobasher2013,vanderbeck2000}.
Кроме того, интеграция раскроя с~календарным планированием затрудняет формализацию в~рамках линейных моделей \cite{gao_energy_2019,keskinocak2002}.
В~обзорах \cite{melega2018review,yanasse2007integrated} отмечается, что интегрированные постановки, объединяющие раскрой, выпуск полуфабрикатов и~календарное планирование, остаются мало исследованными, а~имеющиеся методы, как правило, ориентированы на~упрощённые модели с~одной машиной или без учёта складских ограничений; смежные общие обзоры по cutting and packing и метаэвристикам приведены также в \cite{scheithauer2017_cutpack_optimization, gendreau2019_handbook_metaheur}.
 
 В~связи с~этим в~работе применяется генетический алгоритм, развивающий подход, предложенный в~\cite{klimenko2024_prin,klimenko2025ivdon,klimenko2025fruct} и~расширенный для~учёта ограничений по~срокам, плотности, маркам бумаги, складам, слоям и~БРС.
 Научная новизна разработанного алгоритма определяется не~самим генетическим поиском (являющимся общеизвестной метаэвристикой \cite{goldberg1989genetic, holland1992_adaptation, eiben2015_intro_ec}), а~его адаптацией к~интегрированной модели $G=(\overline{\overline{X}},\overline{\overline{Y}},\overline{\overline{Z}})$, включающей многомашинную систему с~двумя типами оборудования (БДМ и~ПРС/БРС), промежуточные склады с~ограничениями ёмкости, многослойные заказы и~технологический порядок марок бумаги.
Адаптация выражается в~трёх аспектах:
\begin{itemize}
  \item разработка процедур конструктивной генерации допустимых планов (разделы~\ref{sec:alg/cutting_plans}--\ref{sec:alg/random_plan}), обеспечивающих выполнение ограничений модели при~формировании начальной популяции;
  \item проектирование операторов мутации, сохраняющих допустимость и~согласованность компонент $\overline{\overline{X}}$, $\overline{\overline{Y}}$, $\overline{\overline{Z}}$ (раздел~\ref{sec:alg/operators});
  \item замена классического кроссинговера парным отбором, исключающая порождение недопустимых потомков при~наличии межкомпонентных зависимостей.
\end{itemize}

Генетические алгоритмы позволяют гибко управлять временем выполнения, при~этом каждая итерация не~ухудшает качество текущего решения \cite{genetics_2,kureichik1998,goldberg1989genetic}.
Это свойство важно для~практического применения: время расчёта ограничивается заранее заданным размером популяции~$P$, а~результат гарантированно не~хуже лучшего плана из~начальной популяции.

Алгоритм оперирует популяцией $S_t=\{G_1,\ldots,G_{|S_t|}\}$ глобальных планов.
Начальная популяция $S_0$ размера~$P$ формируется вызовами $\mathrm{RandomGlobalPlan}$.
На~каждой итерации выполняются два этапа: (1)~глобальная мутация каждой особи и (2)~парный отбор, уменьшающий размер популяции вдвое.
Итерации продолжаются до~$|S_t|=1$.

Поскольку $\mathrm{GlobalMutate}$ не~ухудшает решение, а~$\mathrm{Crossingover}$ выбирает лучшее из~пары, итоговый план $S_{t,1}$ не~хуже любого плана из~начальной популяции~$S_0$.

Число итераций основного цикла составляет $\lceil\log_2 P\rceil$, так~как размер популяции уменьшается вдвое на~каждом шаге.
На~$k$-й итерации обрабатывается $\lceil P/2^k\rceil$ особей.
Суммарное число вызовов $\mathrm{GlobalMutate}$:
\[
\sum_{k=0}^{\lceil\log_2 P\rceil-1}\left\lceil\frac{P}{2^k}\right\rceil \le 2P.
\]
Каждый вызов $\mathrm{GlobalMutate}$ имеет сложность $O(R+T_X+T_Z)$.
Инициализация популяции требует $P$ вызовов $\mathrm{RandomGlobalPlan}$.
Итоговая сложность алгоритма:
\begin{equation}
    T_{\mathrm{GS}} = O\!\bigl(P\cdot(R\cdot J\cdot n^c_{\max}+T_X+T_Z)\bigr),
    \label{eq:genetic_search_complexity}
\end{equation}
где слагаемое $R\cdot J\cdot n^c_{\max}$ соответствует генерации начальной популяции, а~$T_X+T_Z$~--- пересчёту планов производства и~расписания при~мутациях.
При~фиксированных параметрах оборудования сложность линейна по~$P$ и~$R$.

\textbf{Пример.}
Пусть начальная популяция $S_0=\{G_1,G_2,G_3,G_4\}$ из~$P=4$ планов имеет значения целевой функции:
$\mu(G_1)=\langle 120,\,5,\,12\rangle$,
$\mu(G_2)=\langle 104,\,4,\,10\rangle$,
$\mu(G_3)=\langle 110,\,4,\,8\rangle$,
$\mu(G_4)=\langle 104,\,4,\,11\rangle$.

\emph{Итерация~$t=0$ ($|S_0|=4$).}
Глобальная мутация: для каждого $G_i$ вызывается $\mathrm{GlobalMutate}$.
Допустим, $G_1$ улучшился до~$\mu(G'_1)=\langle 118,\,5,\,11\rangle$, а~остальные не~изменились (мутация не~ухудшила $\mu$).
Парный отбор:
$\mathrm{Crossingover}(G'_1,G_2)$: $\omega(G'_1)=118>104=\omega(G_2)$, выбирается~$G_2$;
$\mathrm{Crossingover}(G_3,G_4)$: $\omega(G_3)=110>104=\omega(G_4)$, выбирается~$G_4$.
Новая популяция: $S_1=\{G_2,G_4\}$.

\emph{Итерация~$t=1$ ($|S_1|=2$).}
Глобальная мутация: допустим, $G_4$ улучшился до~$\mu(G'_4)=\langle 104,\,4,\,9\rangle$.
Парный отбор:
$\mathrm{Crossingover}(G_2,G'_4)$: $\omega(G_2)=\omega(G'_4)=104$, $\varphi(G_2)=\varphi(G'_4)=4$, $\tau(G_2)=10>9=\tau(G'_4)$, выбирается~$G'_4$.
Итог: $S_2=\{G'_4\}$, $|S_2|=1$; алгоритм завершается.
Результат: $\mu(G'_4)=\langle 104,\,4,\,9\rangle$~--- план не~хуже любого из~начальной популяции по~всем критериям.

\section{Выводы по главе}\label{sec:alg/conclusion}

В~данной главе описаны численные методы решения задачи, сформулированной в~главе~\ref{sec:model/statement}.

\begin{enumerate}
    \item Реализован метод полного перебора со~сложностью $O(J^R)$, используемый как эталон для~задач малой размерности и~средство верификации приближённых методов.
    \item Разработаны алгоритмы формирования компонент глобального плана: инкрементальное построение плана нарезки~$\overline{\overline{Y}}$ ($O(R\cdot n^c_{\max})$), автоматическое назначение тамбуров на~совместимые БДМ~--- план производства~$\overline{\overline{X}}$ ($O(N_Y\cdot L_{\max}\cdot I)$) и~операций симуляция для~построения расписания~$\overline{\overline{Z}}$ с~учётом складских ограничений ($O(N_{\mathrm{ops}}\cdot(I+J+\log N_{\mathrm{ops}}))$).
    Данные алгоритмы обеспечивают согласованное формирование всех трёх компонент интегрированной модели, что выходит за~рамки классических методов раскроя \cite{gilmore1961linear,haessler1991}, не~учитывающих производственное планирование и~складскую логистику.
    \item Предложен алгоритм формирования случайного допустимого плана, обеспечивающий разнообразие начальной популяции при~сохранении допустимости всех ограничений модели.
    \item Разработан генетический алгоритм с~операторами локальной и~глобальной мутации и~парным отбором, адаптированными к~многокомпонентной структуре глобального плана $G=(\overline{\overline{X}},\overline{\overline{Y}},\overline{\overline{Z}})$.
    Оператор глобальной мутации обеспечивает согласованность всех компонент плана; парный отбор гарантирует монотонное неухудшение решений.
    Адаптация генетического алгоритма к~интегрированной модели, включающей многомашинную систему с~промежуточными складами, многослойные заказы, БРС и~технологический порядок марок бумаги, составляет научную новизну предложенного подхода.
    \item Итоговая сложность генетического поиска составляет $O(P\cdot(R\cdot J\cdot n^c_{\max}+T_X+T_Z))$, что линейно по~размеру популяции и~числу рулонов, и~позволяет контролировать время расчёта через параметр~$P$.
\end{enumerate}
